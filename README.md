# NutriPlan

iOS-приложение для планирования рациона питания на основе пищевой ценности продуктов.

## О проекте

NutriPlan — это мобильное приложение, которое помогает переводить цель питания в понятные практические действия. Пользователь задаёт параметры профиля и цель, после чего приложение формирует план питания, рассчитывает КБЖУ, подбирает блюда, создаёт список покупок и позволяет учитывать фактическое питание.

Проект разрабатывается как дипломное приложение и ориентирован не только на подсчёт калорий, но и на полный цикл планирования:

**цель → план питания → список покупок → фактическое питание → корректировка рекомендаций**

## Что уже реализовано

### 1. Профиль пользователя
- ввод пола, возраста, роста, веса и уровня активности;
- выбор цели питания;
- настройка микронутриентного фокуса;
- поддержка аллергий, исключаемых продуктов и групп продуктов.

### 2. Расчёт целей питания
- автоматический расчёт суточной калорийности;
- расчёт целевых показателей по белкам, жирам и углеводам.

### 3. План питания
- автоматическое формирование дневного плана;
- разбивка по приёмам пищи: завтрак, обед, ужин, перекус;
- подбор блюд на основе пищевой ценности;
- альтернативный вариант плана через функцию повторного подбора;
- автоматический добор калорий при сильном дефиците;
- адаптивное увеличение граммовки ингредиентов для приближения плана к цели.

### 4. Работа с блюдами
- просмотр состава блюда;
- просмотр КБЖУ и микронутриентов;
- замена ингредиентов на близкие по пищевой ценности;
- изменение граммовки ингредиентов.

### 5. Дневник питания
- перенос блюда из плана в дневник;
- ручное добавление продукта из каталога;
- расчёт фактических показателей за день;
- сравнение «план / факт»;
- рекомендация на следующий день на основе фактического питания.

### 6. Список покупок
- автоматическое формирование на основе текущего плана;
- агрегирование одинаковых ингредиентов;
- группировка по категориям;
- отметка купленных продуктов.

### 7. Данные приложения
- каталог продуктов в JSON;
- каталог рецептов в JSON;
- поддержка категорий, тегов, аллергенов и групп продуктов.

### 8. Тестирование
- unit-тесты для расчёта целей;
- тесты для построения плана;
- тесты для добора калорий;
- тесты для замен;
- тесты для ручного дневника;
- тесты для списка покупок;
- тесты для ограничений и сравнения «план / факт».

## Архитектура интерфейса

Приложение построено на нижней панели навигации из четырёх вкладок:

- **Сегодня** — обзор текущего дня, фактические показатели, быстрые действия;
- **План** — работа с рационом и блюдами;
- **Покупки** — список продуктов на основе плана;
- **Профиль** — персональные данные и ограничения.
