NutriPlan

iOS-приложение для планирования рациона питания на основе пищевой ценности продуктов.

О проекте

NutriPlan — это дипломный iOS-проект, который помогает переводить цель питания в понятные практические действия. Пользователь заполняет профиль, выбирает цель, после чего приложение рассчитывает КБЖУ, формирует план питания, собирает список покупок, позволяет учитывать фактическое питание и адаптировать следующие рекомендации.

Ключевая идея проекта:

цель пользователя → план питания → список покупок → фактическое питание → корректировка следующих рекомендаций.

Что реализовано
1. Профиль пользователя
ввод пола, возраста, роста, веса и уровня активности;
выбор цели питания;
настройка фокуса по микронутриентам;
поддержка аллергий, исключаемых продуктов и исключаемых групп продуктов;
сохранение и обновление профиля через отдельный экран профиля.
2. Расчёт целей питания
автоматический расчёт суточной калорийности;
расчёт целевых значений по белкам, жирам и углеводам;
использование персональных параметров пользователя как основы для построения рациона.
3. План питания
автоматическое формирование дневного плана;
разбиение на приёмы пищи: завтрак, обед, ужин и перекус;
повторный подбор плана;
оптимизация граммовок ингредиентов;
добор калорий перекусами при выраженном дефиците;
просмотр детальной информации по каждому блюду.
4. Работа с блюдами и ингредиентами
просмотр состава блюда;
просмотр КБЖУ и микронутриентов;
изменение граммовки ингредиентов;
замена ингредиентов на близкие по пищевой ценности альтернативы;
объяснение, почему блюдо попало в план.
5. Дневник питания и сравнение «план / факт»
перенос блюда из плана в дневник;
учёт фактического питания;
расчёт фактических показателей за день;
сравнение плана и факта;
рекомендации на следующий день с учётом отклонений.
6. Список покупок
автоматическое формирование на основе текущего плана;
агрегирование одинаковых ингредиентов;
группировка по категориям;
отметка купленных продуктов;
фильтрация, поиск и быстрые действия по списку.
7. Авторизация, аккаунт и синхронизация
стартовый экран приложения;
экран авторизации;
раздел аккаунта;
ручная синхронизация профиля и данных дня;
стартовый маршрут: intro → авторизация / основное приложение / онбординг в зависимости от состояния аккаунта и профиля.
8. Локальные данные и база контента
каталог продуктов хранится в foods.json;
каталог рецептов хранится в recipes.json;
данные включают категории, теги, группы продуктов, аллергенные признаки и пищевую ценность;
структура проекта выделяет отдельные папки Data, Domain, Features и Resources/Data.
9. Тестирование

В репозитории сейчас есть два тестовых направления:

NutriPlanTests — основной старый набор тестов, разложенный по папкам Core, Diary, Planning, Shopping, Support;
NutriPlanStableTests — новый стабильный набор тестов, разложенный по папкам Core, Planning, Shopping.

На текущем этапе в проекте покрываются тестами как минимум следующие блоки:

расчёт целей питания;
список покупок;
замены;
микронутриентная логика;
сравнение плана и факта;
корректировка следующего дня;
суммирование нутриентов;
добор и оптимизация плана.
Архитектура интерфейса

Основной пользовательский интерфейс построен вокруг 4 вкладок нижней панели:

Сегодня — обзор текущего дня, план, факт, микронутриенты, быстрые действия;
План — работа с рационом и блюдами;
Покупки — список продуктов на основе плана;
Профиль — персональные параметры, ограничения и настройки.

Дополнительно приложение использует:

стартовый экран;
экран авторизации;
экран онбординга;
экран аккаунта;
экран синхронизации.

Как запустить проект
Открой NutriPlan.xcodeproj в Xcode.
Выбери нужный симулятор или устройство.
Собери и запусти приложение.
Как запускать тесты
Основной вариант
Product → Test
или Cmd + U
Рекомендуемый вариант для текущего состояния

Использовать схему с NutriPlanStableTests, если нужен быстрый и стабильный прогон новых тестов. Это особенно удобно, если старые тесты из NutriPlanTests временно неактуальны или находятся в процессе обновления.

Статус проекта

Проект находится на продвинутом этапе дипломной разработки: реализовано основное ядро логики, пользовательские сценарии, база продуктов и рецептов, интерфейс приложения и стабильный набор unit-тестов. Дальнейшие шаги — финальная полировка, ручной UX-прогон, доработка тестового покрытия и подготовка дипломной записки и графических материалов.
